# Color

Petrichor's color system is built in OKLCH — a perceptually uniform color space where equal numeric steps produce equal visual steps. This means a `0.12` chroma value looks equally saturated whether the hue is blue, rose, or teal. It also means lightness values are reliable for contrast ratios without the surprises you get from HSL.

The system is **dark-first**. Dark mode is the default in `base.css`. Light mode is a handwritten override layer in `light.css`.

### Error vs Danger

These are semantically distinct — don't conflate them.

**Error** is reactive — something went wrong that the user needs to know about. Form validation failure, network request error, required field left empty. The user made a mistake or the system failed.

**Danger** is proactive — the user is about to do something with a permanent consequence. Delete an account, revoke an API key, submit an irreversible transfer. The user hasn't done anything wrong. You're warning them before they proceed.

Using error colors for a delete button is wrong. Using danger colors for a failed form submission is wrong.

## How theming works

All color tokens live in CSS custom properties on `:root`. Switching brand or color scheme sets `data-brand` and `data-color-scheme` on the `<html>` element — the cascade does the rest.

```css
/* base.css — dark defaults */
:root {
	--ptr-color-brand-primary: oklch(46% 0.12 240); /* rain blue */
	--ptr-color-bg-page: oklch(14% 0.01 258);
}

/* brand.dusk-rose.css — brand override */
[data-brand='dusk-rose'] {
	--ptr-color-brand-primary: oklch(44% 0.13 355); /* rose */
}

/* light.css — light mode override */
[data-color-scheme='light'] {
	--ptr-color-bg-page: oklch(98% 0.01 80);
}
```

Components reference `var(--ptr-color-brand-primary)` and receive whatever value the cascade resolves to. **No component ever knows which brand or theme it's in.**

## Philosophy

**Stormy neutrals with rainbow moments.** The neutral palette leans cool and desaturated — slightly blue-grey, like the sky before rain. Color appears intentionally in brand moments, status signals, and the accent palette. The contrast between muted neutrals and saturated accents gives the system its character.

**OKLCH throughout.** The base scale uses OKLCH for perceptual uniformity. Equal lightness steps look equal. Equal chroma steps look equal. This makes the scale predictable and accessible — you can reason about contrast ratios numerically rather than squinting at swatches.

**Dark first.** The base is dark. Light mode is an explicit override. This reflects where Petrichor lives — data dashboards and late-night writing sessions both want dark mode by default.
