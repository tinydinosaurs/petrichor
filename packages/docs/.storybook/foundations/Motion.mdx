# Motion

Motion in Petrichor is functional, not decorative. Transitions provide feedback, guide attention, and create continuity between states. The system is intentionally restrained — animation serves the interface, never the other way around.

---

## Duration scale

Duration controls how long a transition takes. Shorter durations feel snappy and responsive. Longer durations feel deliberate and dramatic. Most UI interactions use the fast end of the scale.

**The 200ms rule:** If you're unsure which duration to use, start with `base` (200ms). It's the Goldilocks zone — fast enough to feel responsive, slow enough to be perceivable. Adjust from there based on the element's size and importance.

**Ease-out is the default.** Elements entering the viewport or responding to interaction should use ease-out — it feels like the interface is eager to respond. Ease-in is reserved for exits, where the element accelerates away.

## Reduced motion

Always respect `prefers-reduced-motion`. Users who enable this setting may experience motion sickness, vestibular disorders, or simply prefer less movement.

```css
@media (prefers-reduced-motion: reduce) {
	*,
	*::before,
	*::after {
		animation-duration: 0.01ms !important;
		animation-iteration-count: 1 !important;
		transition-duration: 0.01ms !important;
	}
}
```

This is included in `base.css`. When reduced motion is enabled, all transitions become effectively instant — the state change still happens, but without perceived animation.

**Never disable this.** Motion preferences are an accessibility requirement, not a nice-to-have.

---

## Philosophy

**Functional, not decorative.** Every animation should answer: "What does this help the user understand?" If the answer is "nothing, it just looks cool," remove it.

**Fast by default.** The difference between 150ms and 250ms feels negligible when you're watching — but it compounds across hundreds of interactions per session. Err on the faster side.

**Consistency over creativity.** Using the same transition for all dropdowns creates predictability. Varying animations per component creates cognitive overhead. Save creative motion for the moments that deserve it.
